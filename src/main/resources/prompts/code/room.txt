Room rules:
- Use @Entity for database tables.
- Use @PrimaryKey for primary keys.
- For one-to-many relationships, put the foreign key on the child entity.
- Use ForeignKey on the child entity when defining parent-child relationships.
- Add an Index on foreign key columns when appropriate.
- Use @Embedded for the parent object in relationship wrapper classes.
- Use @Relation for child collections.
- Use @Transaction on DAO queries that return relationship objects.
- Do not mutate val properties after object creation.
- If an inserted parent id must be assigned to child items, create copied child objects with the new parent id.
- Keep imports valid for Kotlin. Do not place imports in the middle of a single Kotlin file.
- Prefer simple Room-compatible types unless converters are requested.
